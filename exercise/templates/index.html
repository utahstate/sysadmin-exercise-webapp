USU Sysadmin Exercise

If you see this page then you have completed part 1 of the exercise. For part 2, use a shell on a Linux machine (you can use a shell in this Docker container if you want).

Choose 3 of these exercises and run through them. Please be prepared to share with us your findings and walk us through your solutions.

Linux Exercises

1) Report Users with UID >= 1000

    You are asked to produce a quick report of users on the system that have a UID >= 1000. The output requested is login, UID, and shell. Please show how you would produce this information using a shell script.

2) Write a "Watchdog" Script

    Write a "watchdog" one-line script to monitor the growth of the /var and /home filesystems without using the "watch" command.

3) Remove Invalid Character

    You are looking at a text file that you need to prepare for upload into a database, but it fails due to an invalid character. All lines end with the ^M character. Can you devise a script to remove this character so the file can be loaded into the database?

4) Testing a Condition

    You are writing a shell script and having problems with the following condition: if [ -s $FILE ]. The script is giving you problems and is rather lengthy with multiple similar checks. You want to test this condition on the command line outside of the script.
    Please give a detailed explanation of how to test this condition.

Python Exercixses

Use Python >= 3.11 to complete these exercises

5) Parse a log file with Python

    The objective is to pull log files from a GitHub repository and parse them using Python to find the number of failed authentication attempts in the log file.
    
    Use curl or wget and download the log file at this url:
    https://github.com/logpai/loghub/blob/master/Linux/Linux_2k.log

    Now that you have the log file, use Python to parse it and find the number of failed authentication attempts.

6) Parse a csv file with Python

    Using python, download the CSV file from the following link: https://docs.google.com/spreadsheets/d/e/2PACX-1vR_SfJ7xUN3pcnRw3Z5b0DeINfFmZvWvquoJuBNS0UiA4h2xeftVlOml2BI162JiarV0XyvIOxaEJFQ/pub?gid=0&single=true&output=csv
    Parse the CSV and extract user information.
    Optional:
        Audit the data, making sure that each user meets the following criteria
    	Email addresses must contain "@" and a domain.
    	The role should be either "admin" or "user".
    	The status should be either "active" or "inactive".
    	The created_on date must be in the format YYYY-MM-DD.
        Show the total active users, total inactive users, and a list of invalid entries (users with invalid email, role, status, or date). If any entries are invalid, the list of invalid entries should include the invalid rows.

7) Interact with an API with Python


    Grab 10 random jokes from the official joke API (https://official-joke-api.appspot.com/jokes/random/10).
    Parse the JSON to grab the "programming" type jokes.
    Optional:
      Store these jokes until there are 5 programming jokes.
      Present the jokes only showing the setup and punchline.
      Repeat steps 1-4 three times.
